#️⃣ 5. User Stories (10–15 User Stories)

Below are 15 fully written, professional user stories.

US01 — User Registration

User Story
As a new user, I want to register with my personal information so that I can create a bank account and access online services.

Acceptance Criteria

User must enter full name, email, phone, password.

User receives verification email or OTP.

Registration succeeds only if data is valid.

Duplicate email is not allowed.

US02 — User Login with 2FA

User Story
As a customer, I want to log in securely so that I can access my banking dashboard.

Acceptance Criteria

User enters email + password.

System sends OTP.

Login succeeds only after OTP verification.

US03 — View Account Balance

User Story
As a customer, I want to view my account balance so that I can know how much money I have.

Acceptance Criteria

Dashboard displays current balance.

Balance updates in real time.

US04 — Deposit Funds

Acceptance Criteria:

User selects account.

User enters deposit amount.

System updates balance instantly.

US05 — Withdraw Funds

Acceptance Criteria:

User cannot withdraw more than current balance.

System logs the transaction.

US06 — Internal Transfer (Within same bank)

Acceptance Criteria:

User enters recipient account number.

System validates recipient.

OTP is required before confirming transfer.

US07 — External Transfer (Other Banks)

Acceptance Criteria:

User enters IBAN + bank name.

System performs fraud detection.

Transfer may take 1–2 business days.

US08 — OTP Verification

Acceptance Criteria:

OTP expires in 60 seconds.

Incorrect OTP produces an error.

US09 — Download Account Statement

Acceptance Criteria:

User can select date range.

Statement can be downloaded as PDF.

US10 — Loan Application

Acceptance Criteria:

User enters salary, employment status.

System displays eligibility.

US11 — View Loan Status

Acceptance Criteria:

User can see approved / rejected / pending status.

US12 — Transaction History

Acceptance Criteria:

Shows all transactions with filters.

US13 — Profile Management

Acceptance Criteria:

User can update email, phone, and address.

US14 — Change Password

Acceptance Criteria:

User must enter old password.

New password must meet security rules.

US15 — Admin: Approve Loans

Acceptance Criteria:

Admin sees list of pending loans.

Admin can approve/reject with comments.
